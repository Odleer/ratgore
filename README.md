# RATBITE

![logo](https://github.com/user-attachments/assets/d098742b-8ea1-4606-99fa-385bf988123e)

Это основной репозиторий сервера Ratbite. Ratbite — это форк Hullrot, которая ставит во главу угла персистентность (постоянство мира), бои на кораблях, погружение и более серьёзный тон по сравнению с другими серверами SS14.

Что касается лицензирования, то любой код в папке Hullrot распространяется под лицензией AGPL. Файлы с альтернативными лицензиями помечаются соответствующими комментариями в начале.

Большинство спрайтовых ассетов в папке Hullrot являются проприетарными: это означает, что их нельзя распространять, перелицензировать, использовать и изменять. Папки с портированным контентом являются исключением из этого правила. Другие исключения явно указаны в json-файлах ассетов RSI.

## Участие в разработке
Перед тем как внести свой вклад, рекомендуется проконсультироваться с Главным сопровождающим (Head Maintainer) или Руководителем проекта (Project Lead) в Discord-сервере Hullrot.

Если вы новичок в программировании для SS14 в целом, вам рекомендуется ознакомиться с каналом `#ss14-coding-crashcourse`.

Если вы опытный разработчик или хотите работать над заранее одобренным контентом, у нас есть список доступный в канале `#dev-roadmaps`.

Мы не так строги к стандартам кодирования, как другие серверы SS14. Наша методика проверки учитывает, насколько "производительным" что-то должно быть, однако несколько аспектов следует иметь в виду:
- любой код, значимый для производительности, должен быть производительным;
- от вас ожидается написание безопасного кода;
- от вас ожидается ТЕСТИРОВАНИЕ вашего собственного кода и обеспечение его работоспособности на разумном уровне;
- не оставляйте старый код или код без функциональности;
- от вас ожидается указание источника, если вы портируете контент, через комментарий в файле (для кода) или в `meta.json` (для RSI);
- не смешивайте несвязанные изменения баланса с обычными PR (запросами на включение кода).

### Зависимости для сборки

> - Git
> - .NET SDK 9.0.101

### Windows

> 1. Клонируйте этот репозиторий
> 2. Запустите в терминале `git submodule update --init --recursive`, чтобы загрузить движок
> 3. После внесения любых изменений в исходный код запустите `Scripts\bat\buildAllRelease.bat`
> 4. Запустите `Scripts\bat\runQuickAll.bat`, чтобы запустить клиент и сервер
> 5. Подключитесь к localhost в клиенте и играйте

### Linux / MAC

> 1. Клонируйте этот репозиторий
> 2. Запустите в терминале `git submodule update --init --recursive`, чтобы загрузить движок
> 3. После внесения любых изменений в исходный код запустите `Scripts/sh/buildAllRelease.sh`
> 4. Запустите `Scripts/sh/runQuickAll.sh`, чтобы запустить клиент и сервер
> 5. Подключитесь к localhost в клиенте и играйте
